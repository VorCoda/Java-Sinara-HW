@SpringBootApplication служит для упрощения ввода базовых аннотаций(которые в свою очередь обеспечивают автоматическое написание одних и тех же частей кода для конфигурации, сканирования, регистрации бинов и импорта доп. классов конфигурации), она содержит в себе следующие три аннотации:\
@EnableAutoConfiguration - включает автоматическую настройку Spring ApplicationContext путем сканирования компонентов пути к классам и регистрации бинов, соответствующих различным условиям \
@ComponentScan - активирует сканирование аннотации @Component для пакета, в котором находится приложение \
@Configuration - указывает на то, что объект является источником определений бина

Параметры аннотации @SpringBootApplication:

scanBasePackages - надо указать список пакетов для сканирования

scanBasePackageClasses - список отдельных классов для сканирования

exclude - список классов автоконфигурации, которые нужно исключить

excludeName - то же, что exclude, но передаём строки

nameGenerator - генератор имён для бинов. Должен реализовывать интерфейс BeanNameGenerator, по умолчанию используется реализация AnnotationBeanNameGenerator

proxyBeanMethods - касается бинов, объявленных через метод (метод с @Bean в классе с @Configuration). По умолчанию true, то есть, спринг проксирует вызовы этого метода и возвращает уже созданный при запуске объект./
Если false, то спринг не проксирует запросы и при вызове метода создаётся каждый раз новый объект.
